- Satisfy the skill requirements listed in [JD-001](../JD/JD-001.md)

- Use deidentified data during testing and demonstrations

## Data Strategy

- RAGAS evaluation requires a sufficient volume of deidentified data
- Key considerations: data sources, persistence, evaluation workflow, and demo presentation
- All sensitive information must be masked before ingestion (`[REDACTED]` pattern)

## Observability

- LangSmith free tier may be insufficient → using **LangFuse** instead (local Docker + optional cloud deployment)
- All LLM calls, retrieval calls, and agent steps must be traced via LangFuse

## Persistence

- Each retrieval operation's data (query, top-K results, relevance scores) should be persisted to a database (e.g., Azure CosmosDB) for evaluation, replay, and analysis
