# Agent 开发场景设计

本文档涵盖了从个人简历到复杂任务的三个 Agent 演进场景，旨在展示 Agent 如何从被动问答转向主动引导、自我修正及复杂推理。

---

## 场景一：个人简历深度访谈 Agent
该场景的核心在于 Agent 不再是被动回答，而是能够**主动引导**和**自我修正**。

### 1. 行为逻辑
* **意图识别**：能够从简历中识别多段相关经历以应对复杂的提问（如：关于大数据的经验）。
* **工具调用 (Tool Use)**：调用 `Skill_Analyzer` 等工具，对比简历内容与职位的匹配度，而非直接生成回答。
* **多步推理**：自主决定回答的优先级。例如，先讲解技术最契合的经历，再根据对方兴趣调取规模更大的项目数据。
* **反向提问**：这是 Agent 的核心。回答后能根据上下文（如处理量要求）主动询问用户是否需要了解更多技术细节（如 Flink 优化）。

### 2. 可挖掘功能
* **Memory（记忆）**：记住面试官之前提到的痛点，在后续回答中反复强调匹配性。
* **Reflection（反思）**：当收到负面反馈时，自动搜索库中其他相关的隐藏技能进行补偿回答。

## 场景二：全自动猎头助手（推荐面试使用）
此场景能够将 **LangChain（编排）**与 **LlamaIndex（数据）**的优势发挥到极致。

### 1. 场景描述
给定一份“职位描述 (JD)”，Agent 自动在“简历库”中检索最合适的人选并撰写推荐信。

### 2. 为什么需要 Agent
* **规划 (Planning)**：第一步先拆解 JD 并提取关键词。
* **执行 (Execution)**：调用 `Retriever` 工具寻找候选人。
* **循环 (Loop)**：若匹配度低于 70%，自主决定扩大搜索范围（如将“Java”扩大至“所有 JVM 语言”）并重新检索。
* **工具集成**：调用 `Salary_Calculator` 外部 API 计算薪资匹配度，最后通过 `Email_Generator` 生成信件。

### 3. 可挖掘功能
* **Self-Correction（自纠错）**：若生成的信件存在事实错误，Agent 会再次检索原简历进行校对。

## 场景三：私人技术债务/文档扫描 Agent
这是一个偏向工具类的场景，用于展示对复杂数据结构的处理能力。

### 1. 场景描述
将过去所有项目的代码片段、技术文档、报错日志输入系统，让 Agent 分析技术思路随时间的变化（如：过去三年处理高并发问题的思路演进）。

### 2. 为什么需要 Agent
* **多文档对比 (Comparison)**：跨越不同年份的文档和代码进行横向对比。
* **综合总结**：并非简单的 RAG 拼接，而是运行 `Summarizer_Agent` 进行语义差异分析。
* **深度挖掘**：主动探究技术变更背后的原因。例如，通过查阅当年的报错日志，分析为何从 Redis 锁改为分布式数据库原生锁。

---

## 总结：RAG vs. Agent

| 特性 | 普通 RAG | Agent |
| :--- | :--- | :--- |
| **任务目标** | 找寻特定信息（如“找张三的简历”） | 判断综合匹配度（如“张三适合这个岗位吗？”） |
| **执行流程** | 检索 -> 输出 -> 结束 | 检索简历及要求 -> 思考缺项 -> 决策调取外部信息补全 -> 给出最终结论 |

---

## 实施建议
针对“个人面试库”项目，建议先从以下三个 **Tools** 开始构建：

1.  **Query_Resume (RAG)**：用于查询基础事实。
2.  **Compare_Experience (Comparison)**：用于对比两段经历的异同。
3.  **Career_Consultant (Strategy)**：基于经历为面试者提供反向提问策略。

通过 **LangGraph** 或 **LangChain Agent** 将这些工具串联，系统将演变成一个有灵魂的“求职替身”。
